# セッション

> 要約: Pi は会話をセッションとして保存し、途中再開、分岐、別経路の比較をできるようにしています。このページはセッション保存場所、主要コマンド、`/tree` による分岐操作、ブランチ要約の扱いをまとめています。

Pi は会話をセッションとして保存するため、作業を継続したり、以前のターンから分岐したり、過去の経路を再訪したりできます。

## セッション保存

セッションは `~/.pi/agent/sessions/` に自動保存され、作業ディレクトリごとに整理されます。各セッションはツリー構造を持つ JSONL ファイルです。

```bash
pi -c                  # もっとも新しいセッションを続行
pi -r                  # 過去のセッションを閲覧して選択
pi --no-session        # 一時モード; 保存しない
pi --name "my task"    # 起動時にセッション表示名を設定
pi --session <path|id> # 特定のセッションファイルまたは部分セッション ID を使う
pi --fork <path|id>    # セッションファイルまたは部分セッション ID を新しいセッションへ fork
```

対話モードでは `/session` を使うと、現在のセッションファイル、セッション ID、メッセージ数、トークン数、コストを確認できます。

JSONL のファイル形式と SessionManager API については [Session Format](session-format.md) を参照してください。

## セッションコマンド

| Command | Description |
|---------|-------------|
| `/resume` | 以前のセッションを閲覧して選択 |
| `/new` | 新しいセッションを開始 |
| `/name <name>` | 現在のセッション表示名を設定 |
| `/session` | セッション情報を表示 |
| `/tree` | 現在のセッションツリーを移動 |
| `/fork` | 以前のユーザーメッセージから新しいセッションを作成 |
| `/clone` | 現在のアクティブブランチを新しいセッションへ複製 |
| `/compact [prompt]` | 古いコンテキストを要約する。[Compaction](compaction.md) を参照 |
| `/export [file]` | セッションを HTML へエクスポート |
| `/share` | 非公開 GitHub gist と共有用 HTML リンクとしてアップロード |

## セッションの再開と削除

`/resume` は現在のプロジェクト向けの対話的セッションピッカーを開きます。`pi -r` でも起動時に同じピッカーが開きます。

ピッカーでは次の操作ができます。

- タイプして検索
- `Ctrl+P` でパス表示を切り替え
- `Ctrl+S` でソートモードを切り替え
- `Ctrl+N` で名前付きセッションだけに絞り込み
- `Ctrl+R` でリネーム
- `Ctrl+D` で削除し、その後確認

利用可能な場合、pi はファイルを完全削除する代わりに `trash` CLI を使います。

## セッション名を付ける

人間が読みやすいセッション名を付けるには `/name <name>` を使います。

```text
/name Refactor auth module
```

起動時には `--name` または `-n` で設定できます。

```bash
pi --name "Refactor auth module"
pi --name "CI audit" -p "Review this build failure"
```

名前付きセッションは `/resume` や `pi -r` で見つけやすくなります。

## `/tree` による分岐

セッションはツリーとして保存されます。各エントリには `id` と `parentId` があり、現在位置はアクティブな leaf です。`/tree` を使うと、新しいファイルを作らずに任意の過去地点へジャンプしてそこから続行できます。

<p align="center"><img src="images/tree-view.png" alt="Tree View" width="600"></p>

形状の例:

```text
├─ user: "Hello, can you help..."
│  └─ assistant: "Of course! I can..."
│     ├─ user: "Let's try approach A..."
│     │  └─ assistant: "For approach A..."
│     │     └─ user: "That worked..."  ← active
│     └─ user: "Actually, approach B..."
│        └─ assistant: "For approach B..."
```

### ツリー操作

| Key | Action |
|-----|--------|
| ↑/↓ | 表示中エントリを移動 |
| ←/→ | ページアップ / ページダウン |
| Ctrl+←/Ctrl+→ or Alt+←/Alt+→ | 折りたたみ / 展開、またはブランチ区間間の移動 |
| Shift+L | 選択中エントリにラベルを設定 / 解除 |
| Shift+T | ラベルのタイムスタンプ表示を切り替え |
| Enter | エントリを選択 |
| Escape/Ctrl+C | キャンセル |
| Ctrl+O | フィルタモードを切り替え |

フィルタモードは default、no-tools、user-only、labeled-only、all です。デフォルトは [Settings](settings.md) の `treeFilterMode` で設定できます。

### 選択時の挙動

ユーザーまたはカスタムメッセージを選択した場合:

1. leaf を選択したメッセージの親へ移動する
2. 選択したメッセージ本文をエディタに入れる
3. 編集して再送信し、新しいブランチを作れる

アシスタント、ツール、compaction、その他のユーザー以外のエントリを選択した場合:

1. leaf をそのエントリへ移動する
2. エディタは空のままにする
3. その地点から続行できる

ルートのユーザーメッセージを選択すると、leaf は空の会話にリセットされ、元のプロンプトがエディタへ入ります。

## `/tree`、`/fork`、`/clone`

| Feature | `/tree` | `/fork` | `/clone` |
|---------|---------|---------|----------|
| 出力 | 同じセッションファイル | 新しいセッションファイル | 新しいセッションファイル |
| 表示 | ツリー全体 | ユーザーメッセージ選択 UI | 現在のアクティブブランチ |
| 典型的な用途 | 代替案を同じ場所で試す | 以前のプロンプトから別セッションを始める | 現在の作業を複製してから続ける |
| 要約 | 任意のブランチ要約あり | なし | なし |

代替案をまとめて保持したいなら `/tree` を使います。別のセッションファイルに分けたいなら `/fork` または `/clone` を使います。

## ブランチ要約

`/tree` であるブランチから別のブランチへ切り替えるとき、pi は離れるブランチを要約し、その要約を新しい位置へ付与できます。これにより、離れた経路全体を再生しなくても重要な文脈を保持できます。

プロンプトが表示されたら、次のいずれかを選びます。

1. 要約しない
2. デフォルトプロンプトで要約する
3. 焦点指定付きのカスタム指示で要約する

ブランチ要約の内部実装と拡張フックについては [Compaction](compaction.md) を参照してください。

## セッション形式

セッションファイルは JSONL で、メッセージエントリ、モデル変更、thinking-level 変更、ラベル、compaction、branch summary、拡張機能エントリを含みます。

パーサー、拡張機能、SDK 利用、完全な SessionManager API については [Session Format](session-format.md) を参照してください。
